# DAS Intelligence Skills Master Index

This file is the central map of the `das-intelligence` repository.

## Repository Purpose

`das-intelligence` is a practical AI skills library for Das Experten business work across marketing, websites, SEO, content, marketplaces, sales, operations, logistics, legal support, regulatory preparation, agents, automation, and analytics.

## Default Skill Format

```text
skill-name/
├── SKILL.md
├── templates.md
└── checklist.md
```

Some older folders may contain short fallback files such as `check.md`, `template.md`, `notes.md`, or `status.txt` where the GitHub connector blocked normal filenames.

# Category Selection Guide

- Brand, offer, GTM, pricing, launch, Google Ads → `skills/marketing/`
- Website pages, UX, CRO, checkout, mobile → `skills/website/`
- Organic search, metadata, schema, AI search → `skills/seo/`
- Posts, videos, UGC, email, articles, copy → `skills/content/`
- WB, Ozon, Amazon, TikTok Shop, marketplace cards or ads → `skills/marketplaces/`
- Distributors, export, retail buyers, terms, quotes → `skills/sales/`
- SOPs, vendors, priorities, repeated cases → `skills/operations/`
- Routes, warehouses, stock, landed costs, shipment files → `skills/logistics/`
- Document preparation, agreements, formal letters → `skills/legal/`
- Labels, certificates, registrations, market readiness → `skills/regulatory/`
- Agents, routing, handoffs, director decisions → `skills/agents/`
- Make.com, APIs, syncs, automation errors → `skills/automation/`
- Dashboards, metrics, funnels, ROI, reports → `skills/analytics/`

## Google Ads Selection Guide

- Objective is unclear → `google-ads-performance-director`
- Tracking or values may be wrong → `conversion-value-tracking-auditor`
- Maximize profitable value → `google-ads-roas-optimizer`
- Reduce qualified acquisition cost → `google-ads-cpa-optimizer`
- Reduce inefficient click cost → `google-ads-cpc-quality-score-optimizer`
- Improve Search query quality → `search-campaign-query-optimizer`
- Improve Performance Max → `performance-max-optimizer`
- Choose bidding or allocate budget → `budget-bidding-strategy-planner`
- Test a major change → `google-ads-experiment-planner`

# Maintenance Rules

When adding new skills:

1. Keep the KISS structure.
2. One skill should solve one practical task.
3. Avoid theory-heavy files.
4. Include Purpose, When to Use, Inputs, Workflow, Output, Common Mistakes, and Related Skills.
5. Add templates and checklist.
6. Update the local category README.
7. Update this `MASTER_INDEX.md`.

The master index should remain the fastest way to understand the repository.
