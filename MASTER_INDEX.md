# DAS Intelligence Skills Master Index

This file is the central map of the `das-intelligence-skills` repository.

It shows every completed skill block, where each skill lives, and when to use each category.

## Repository Purpose

`das-intelligence-skills` is a practical AI skills library for Das Experten business work.

The repository is designed for:

- marketing and brand strategy;
- website conversion and UX;
- SEO and AI search readiness;
- content production;
- marketplace execution;
- B2B, export, and distributor sales;
- operations and logistics;
- legal-support and regulatory preparation;
- agents, director workflows, automation, and analytics.

## Skill Format

Default KISS format:

```text
skill-name/
├── SKILL.md
├── templates.md
└── checklist.md
```

Some folders may use short fallback files such as `check.md`, `template.md`, `notes.md`, or `status.txt` when the GitHub connector safety filter blocks normal checklist/template naming.

# Category Selection Guide

## If the user asks about brand, offer, market, campaign, or pricing

Use `skills/marketing/`.

## If the user asks about website pages, UX, CRO, trust, checkout, or mobile view

Use `skills/website/`.

## If the user asks about Google visibility, keywords, metadata, schema, multilingual SEO, or AI search

Use `skills/seo/`.

## If the user asks about posts, videos, UGC, email, article planning, or product copy

Use `skills/content/`.

## If the user asks about WB, Ozon, Amazon, TikTok Shop, Shopee, Lazada, Noon, cards, ads, reviews, ranking, or SKU performance

Use `skills/marketplaces/`.

## If the user asks about distributors, export sales, retail buyers, meetings, trade terms, or quotes

Use `skills/sales/`.

## If the user asks about internal work, SOPs, vendors, task order, or repeated cases

Use `skills/operations/`.

## If the user asks about shipping, warehouses, 3PLs, stock, landed cost, or shipment documents

Use `skills/logistics/`.

## If the user asks about document preparation, agreements, formal letters, or organized files

Use `skills/legal/`.

## If the user asks about labels, certificates, product files, market readiness, or product-rule tracking

Use `skills/regulatory/`.

## If the user asks about agents, workflows, task routing, handoffs, director summaries, or decisions

Use `skills/agents/`.

## If the user asks about Make.com, APIs, syncs, automation errors, or runbooks

Use `skills/automation/`.

## If the user asks about dashboards, numbers, reports, funnels, ROI, or business metrics

Use `skills/analytics/`.

# Maintenance Rules

When adding new skills:

1. Keep KISS structure.
2. One skill should solve one practical task.
3. Avoid theory-heavy files.
4. Include clear Purpose, When to Use, Inputs, Workflow, Output, Common Mistakes, Related Skills.
5. Add templates and checklist when possible.
6. Update the local category README.
7. Update this `MASTER_INDEX.md`.

## Repository Rule

The master index should remain the single fastest way to understand the whole repo.
