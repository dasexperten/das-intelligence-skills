# Value Proposition Architect Checklist

- [ ] Target customer is specific.
- [ ] Main problem or desire is clear.
- [ ] Features are translated into benefits.
- [ ] Primary promise is not overloaded.
- [ ] Claims have proof.
- [ ] Message is understandable in 5 seconds.
- [ ] Website, marketplace, ad, and B2B versions are adapted.
- [ ] Objections are answered.
- [ ] Weak claims are removed.
