# Value Proposition Architect

## Purpose

Build a clear value proposition that explains why a customer should choose a product, service, offer, or brand now.

This skill converts features, ingredients, technologies, price, proof, and customer pains into a practical value message for websites, marketplaces, ads, sales decks, and distributor pitches.

## When to Use

Use this skill when the task is to:

- write or improve a value proposition;
- clarify product benefits;
- make a product page stronger;
- create marketplace selling points;
- build a B2B pitch;
- compare customer pain against product proof;
- improve conversion messaging.

## When NOT to Use

Do not use when the task is full brand strategy, detailed pricing, visual design, legal claim validation, or technical SEO.

## Inputs

Useful inputs:

- product name;
- target customer;
- problem or desire;
- product features;
- proof points;
- competitor alternatives;
- price tier;
- sales channel;
- market or country.

## Workflow

1. Identify the customer's main pain, desire, or job-to-be-done.
2. Separate features from benefits.
3. Translate each feature into a customer outcome.
4. Rank benefits by purchase influence.
5. Attach proof to each major claim.
6. Remove weak, generic, or unsupported claims.
7. Write the main value proposition.
8. Create short channel versions for website, marketplace, ads, and B2B.
9. Add objections and counter-messages.

## Output

Always provide:

- customer problem;
- product promise;
- key benefits;
- reasons to believe;
- final value proposition;
- short channel versions;
- objections and answers;
- improvement actions.

## Best Practices

- Lead with customer value, not internal product pride.
- Use proof to support strong claims.
- Make the message understandable in 5 seconds.
- Match the value proposition to the channel.
- Keep one primary promise and 2–4 supporting benefits.

## Common Mistakes

- Listing features without outcomes.
- Saying “innovative” or “premium” without proof.
- Making too many promises at once.
- Ignoring customer objections.
- Using the same value proposition for every market.
