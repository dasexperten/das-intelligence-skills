# UX Audit

## Purpose

Audit user experience across pages, devices, navigation, forms, content hierarchy, and interaction flow to identify usability problems that block understanding, trust, and conversion.

This skill is built for e-commerce, DTC websites, B2B websites, landing pages, product pages, and international websites.

## When to Use

Use this skill when the task is to:

- audit website UX;
- find usability problems;
- improve navigation and page flow;
- review mobile and desktop experience;
- reduce confusion;
- improve content hierarchy;
- make the site easier to use.

## When NOT to Use

Do not use for brand strategy, deep technical SEO, or full analytics tracking setup. Use this skill for user experience diagnosis and practical fixes.

## Inputs

Useful inputs:

- website URL or screenshots;
- target user;
- main tasks;
- device type;
- page list;
- current issues;
- analytics or heatmap signals;
- conversion goal;
- navigation structure.

## Workflow

1. Define the user task and success outcome.
2. Review navigation, hierarchy, readability, and layout.
3. Check mobile and desktop usability.
4. Identify confusing steps, unclear labels, and interaction friction.
5. Review forms, buttons, menus, and page scanning.
6. Identify accessibility and clarity issues.
7. Prioritize fixes by severity and effort.

## Output

Always provide:

- UX diagnosis;
- usability issues;
- navigation fixes;
- layout and hierarchy fixes;
- mobile issues;
- accessibility notes;
- priority roadmap;
- next steps.

## Best Practices

- Review the site from the user's task, not from internal structure.
- Make important actions obvious.
- Keep labels simple and predictable.
- Prioritize mobile usability.
- Reduce cognitive load and unnecessary choices.

## Common Mistakes

- Confusing design preference with UX problem.
- Ignoring mobile users.
- Hiding important actions.
- Using unclear menu labels.
- Making content visually attractive but hard to scan.
