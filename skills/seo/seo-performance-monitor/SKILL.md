# SEO Performance Monitor

## Purpose

Monitor SEO performance over time and explain what changed, why it may have changed, and what actions should be taken next.

## When to Use

Use this skill when the task is to create SEO reports, review organic changes, track rankings, analyze Search Console or GA4 trends, or summarize SEO progress.

## Inputs

Useful inputs: date range, Search Console data, GA4 data, ranking data, priority pages, target queries, content changes, technical changes, campaigns, and external events.

## Workflow

1. Define reporting period and SEO goals.
2. Review clicks, impressions, CTR, positions, organic sessions, conversions, and revenue or leads.
3. Segment by page, query, device, country, and content type.
4. Identify wins, losses, likely causes, and uncertain areas.
5. Recommend actions and next monitoring checkpoints.

## Output

Always provide performance summary, wins, losses, likely causes, priority pages, KPI table, action list, and next checkpoint.

## Common Mistakes

- Reporting metrics without explanation.
- Ignoring segmentation.
- Claiming causes without evidence.
- Focusing only on rankings.
- No next actions.
