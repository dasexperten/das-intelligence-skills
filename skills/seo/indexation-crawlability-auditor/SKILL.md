# Indexation Crawlability Auditor

## Purpose

Diagnose whether important pages can be discovered, crawled, rendered, indexed, and kept in search results.

## When to Use

Use this skill when the task is to check crawlability, indexation, sitemap coverage, robots rules, canonical issues, noindex, crawl budget, or pages missing from search.

## Inputs

Useful inputs: priority page list, sitemap, robots.txt, Search Console indexing data, crawl report, canonical tags, noindex tags, status codes, and internal link structure.

## Workflow

1. Define priority pages that should be indexed.
2. Check discovery through sitemap and internal links.
3. Review robots, status codes, canonicals, noindex, redirects, and duplicates.
4. Separate crawl problems from indexation quality problems.
5. Recommend fixes and validation steps.

## Output

Always provide indexation diagnosis, crawl blockers, index blockers, quality risks, priority page status, fix list, and validation steps.

## Common Mistakes

- Assuming sitemap inclusion means indexation.
- Ignoring internal links.
- Confusing crawled with indexed.
- Fixing low-priority pages before money pages.
