# Site Architecture Planner

## Purpose

Plan SEO-friendly website architecture that helps users and search engines understand categories, products, topics, markets, languages, and priority pages.

## When to Use

Use this skill when the task is to organize website structure, category hierarchy, URL structure, navigation, topic hubs, multilingual structure, or content-to-product paths.

## Inputs

Useful inputs: business model, product categories, target markets, languages, page types, keyword groups, priority pages, current sitemap, navigation, and internal links.

## Workflow

1. Define business goals and priority page types.
2. Group pages by category, topic, market, and user journey.
3. Design hierarchy, URL logic, navigation, breadcrumbs, and internal links.
4. Identify orphan pages, duplicate structures, and crawl depth problems.
5. Recommend rollout and validation steps.

## Output

Always provide architecture diagnosis, recommended structure, URL logic, navigation notes, internal link plan, risks, and next actions.

## Common Mistakes

- Building structure around internal company logic instead of user intent.
- Too many deep pages.
- No hub pages.
- Weak internal linking.
- Mixing languages or markets without clear structure.
