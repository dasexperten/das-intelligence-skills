# Programmatic SEO Planner

## Purpose

Plan scalable SEO page systems that create many useful pages from structured data without thin, duplicate, or low-value content.

## When to Use

Use this skill when the task is to plan location pages, comparison pages, product variants, category pages, use-case pages, or template-based SEO at scale.

## Inputs

Useful inputs: page pattern, data source, target queries, markets, page template, unique data fields, internal links, indexation rules, quality threshold, and business goal.

## Workflow

1. Define page pattern and search demand.
2. Verify there is enough unique value per page.
3. Map required data fields and page modules.
4. Define template, internal links, canonical rules, and indexation rules.
5. Create quality thresholds and rollout plan.
6. Recommend test set before scale.

## Output

Always provide programmatic SEO concept, page pattern, data requirements, template modules, quality rules, indexation notes, rollout plan, and risks.

## Common Mistakes

- Creating thin pages.
- Scaling before testing quality.
- No unique value per page.
- Poor canonical or indexation rules.
- No internal linking structure.
