# Technical SEO Auditor

## Purpose

Find technical SEO issues that prevent search engines from crawling, indexing, understanding, and ranking important pages.

## When to Use

Use this skill when the task is to review crawlability, indexation, robots.txt, sitemap, canonical tags, redirects, status codes, duplicate pages, pagination, performance, or structured data foundations.

## Inputs

Useful inputs: website URL, crawl data, Search Console data, robots.txt, sitemap, CMS/platform, page templates, error reports, and priority pages.

## Workflow

1. Define priority pages and technical risk.
2. Review robots, sitemap, status codes, redirects, canonicals, noindex, and duplicates.
3. Check template-level technical patterns.
4. Review mobile, speed, structured data, and international tags when relevant.
5. Prioritize fixes by indexation and ranking impact.

## Output

Always provide technical diagnosis, crawl/index blockers, template issues, priority fixes, validation steps, and next actions.

## Common Mistakes

- Fixing minor warnings before index blockers.
- Ignoring template-level problems.
- Confusing canonical hints with guaranteed indexation.
- Not validating after fixes.
